# To-Do リストアプリ 開発計画書

機能仕様・データモデル・画面設計などの詳細は [spec.md](spec.md) を参照。本書は開発の進め方（スモールステップ）とマイルストーンを定義する。

## 1. 目的・スコープ

- 個人利用・学習目的のシンプルな to-do リスト管理アプリを作成する。
- まずは最小限の機能（MVP）で動くものを作り、小さいステップで段階的に機能を追加していく。
- 将来的に複数端末での同期やユーザー管理が必要になった場合に拡張できる構成にしておく（ただし最初から作り込みすぎない）。

## 2. 想定ユーザー・利用シーン

- 利用者：自分一人（学習目的も兼ねる）
- 利用シーン：日々のタスク管理、買い物リスト、学習タスクの管理など
- 利用規模：個人利用のため、大規模なアクセスやマルチユーザー対応は考慮しない

## 3. 開発ステップ（スモールステップ）

各ステップは「動くものが確認できる」単位に分割する。

## 4. マイルストーン目安

| マイルストーン | 完了条件 |
|---|---|
| M1: 環境構築完了 | フェーズ0〜1完了、Hello World表示 |
| M2: MVP完成 | フェーズ2〜3完了、ローカルで一通りのCRUDが動作 |
| M3: 公開 | フェーズ4完了、URLでアクセス可能 |
| M4: 機能拡張版 | フェーズ5の主要機能が実装済み |
